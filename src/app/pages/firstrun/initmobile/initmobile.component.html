Iniciar proyecto de Mobile

Paso 1: Clona el repositorio en tu entorno de desarrollo local: 

	
git clone https://github.com/alejandromartincruz/classpip-mobile.git

Paso 2: Instala la versión 2.2.2 de ionic y la versión 6.4.0 de cordova con npm: 

	
npm install -g ionic@2.2.2
npm install -g cordova@6.4.0

Paso 3: Instala las dependencias del proyecto: 

	
cd classpip-mobile
npm install
mkdir www
cordova prepare

Paso 4: Inicia Classpip-mobile (por defecto en http://localhost:8100) con el siguiente comando: 

	
ionic serve --lab

Origen de los datos

Hay que asegurarse de que la aplicación de servicios esta funcionando antes de intentar usar la aplicación de mobile, ya que la aplicación de servicios es el origen y destino de los datos. De no ser así, al intentar iniciar sesión se obtendrá el error: ERROR: Ha ocurrido un error interno en el servidor. Por favor, contacta con tu administrador.

También hay que asegurar que este configurado el origen de los datos en localhost:3000. Para ello hay que dirigirse al archivo 'src/app/app.config.ts' y asegurarse de que el SERVER_URL apunta a localhost:3000 y no a https://api.classpip.com basta con dejar las lineas como se muestra a continuación: 

		// Server connection
// public static get SERVER_URL(): string {{'{'}} return 'https://api.classpip.com'; {{'}'}} // PRO
public static get SERVER_URL(): string {{'{'}} return 'http://localhost:3000'; {{'}'}} // DEV
